### Section 3 – Data Cleaning & Feature Engineering

#### Cell 24 — Section Header *(Markdown)*
"Section 3 – Data Cleaning & Feature Engineering / 3.1 Convert Datetime Columns" — justifies converting string timestamps to `datetime64` for arithmetic operations.

#### Cell 25 — Convert Datetime Columns *(Code)*
Converts all timestamp columns in `orders` and `reviews` from `object` to `datetime64` using `errors='coerce'` (unparse-able values become `NaT` instead of crashing).

```python
datetime_cols_orders = [
    'order_purchase_timestamp', 'order_approved_at',
    'order_delivered_carrier_date', 'order_delivered_customer_date',
    'order_estimated_delivery_date'
]
for col in datetime_cols_orders:
    orders[col] = pd.to_datetime(orders[col], errors='coerce')
```

**Before:** `'2017-10-02 10:56:33'` (string)
**After:** `Timestamp('2017-10-02 10:56:33')` (datetime64)

#### Cell 26 — Subsection Header *(Markdown)*
"3.2 Handle Missing Values (Justified per Column)"

#### Cell 27 — Impute / Drop Missing Values *(Code)*
Applies a column-specific strategy justified by the business impact analysis:

| Column | Strategy | Reason |
|---|---|---|
| `order_id` (null) | **Drop** | Cannot identify the order |
| `product_category_name` | **Fill → `'unknown'`** | Preserves row; flagged for later |
| `product_weight_g`, `product_length_cm`, etc. | **Fill → median** | Small gap (< 0.1%), median is robust |
| `review_comment_message` | **Fill → `''`** | Absent text, not missing data |

```python
products['product_category_name'].fillna('unknown', inplace=True)

dim_cols = ['product_weight_g','product_length_cm',
            'product_height_cm','product_width_cm']
for col in dim_cols:
    products[col].fillna(products[col].median(), inplace=True)

reviews['review_comment_message'].fillna('', inplace=True)
```

#### Cell 28 — Subsection Header *(Markdown)*
"3.3 Standardise Text Fields"

#### Cell 29 — Standardise Product Category Names *(Code)*
1. Strips whitespace and lowercases all `product_category_name` values.
2. Left-joins the English translation from `cat_trans`.

```python
products['product_category_name'] = (
```

### Engineered Features

#### Cell 34 — Feature 1 Definition *(Markdown)*
**Delivery Delay** — difference in days between actual and estimated delivery, with business justification for logistics SLA monitoring.

#### Cell 35 — Feature 1: Delivery Delay *(Code)*
```python
orders['delivery_delay_days'] = (
    orders['order_delivered_customer_date']
    - orders['order_estimated_delivery_date']
).dt.days

orders['is_late_delivery'] = orders['delivery_delay_days'] > 0
```

**Figure 2:** Histogram of delivery delay distribution. Negative = early, positive = late.

#### Cell 36 — Feature 2 Definition *(Markdown)*
**Customer Lifetime Value (CLV)** — total payment value across all orders per customer; supports CRM segmentation strategy.

#### Cell 37 — Feature 2: CLV *(Code)*
```python
# Sum installment rows to get order-level payment
order_pay = payments.groupby('order_id')['payment_value'].sum().reset_index()

# Join to orders → get customer_id
order_cust = orders[['order_id','customer_id']].merge(order_pay, on='order_id')

# Aggregate to customer level
clv_df = order_cust.groupby('customer_id')['payment_value'].sum().reset_index()
clv_df.rename(columns={'payment_value': 'clv'}, inplace=True)
```

**Example output:**
```
CLV statistics (96096 customers):
  mean   BRL 154.10
  median BRL  99.99
  max    BRL 13,664
```

#### Cell 38 — Feature 3 Definition *(Markdown)*
**Purchase Frequency** — count of distinct orders per customer; one component of the RFM (Recency-Frequency-Monetary) framework.

#### Cell 39 — Feature 3: Purchase Frequency *(Code)*
```python
purchase_freq = orders.groupby('customer_id')['order_id'].nunique().reset_index()
purchase_freq.rename(columns={'order_id': 'purchase_count'}, inplace=True)
```

**Example output:**


#### Cell 40 — Feature 4 Definition *(Markdown)*
**Average Basket Value** — mean total value (price + freight) per order; standard retail KPI for revenue tracking.

#### Cell 41 — Feature 4: Average Basket Value *(Code)*
```python
order_items['item_total'] = order_items['price'] + order_items['freight_value']
basket_values = order_items.groupby('order_id')['item_total'].sum().reset_index()
basket_values.rename(columns={'item_total': 'basket_value'}, inplace=True)

print(f"Mean basket  : BRL {basket_values.basket_value.mean():.2f}")
print(f"Median basket: BRL {basket_values.basket_value.median():.2f}")
```

**Revenue per Seller** — total sales revenue attributed to each seller; used for seller partnership and performance ranking.
